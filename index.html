Google Drive API

Pendiente

Para subir archivos a Drive desde GitHub Pages se requiere un Client ID de Google Cloud. Mientras tanto, la app puede guardar copias en una carpeta local sincronizada de Google Drive.

Google Sheet / Excel como fuente

Preparado

En este prototipo la alimentación de datos se guarda localmente y se exporta a Excel actualizado. Para alimentar un Google Sheet real se requiere OAuth de Google Cloud o una integración posterior con Apps Script.

Identidad visual

El prototipo usa el logotipo descargado desde el sitio oficial de GLF y la paleta detectada en su hoja de estilos: azul #1E3A6C, cyan #00C1DE, verde #038C7C, blanco y negro institucional. Si luego se entrega un manual de marca formal, se ajustarán proporciones, usos alternos y restricciones.

Correo de avisos

Configura el destinatario que recibirá los avisos preparados por el prototipo. Para envío automático real se deberá conectar una cuenta institucional mediante Microsoft 365, Google Workspace o un backend.

Manual de uso rápido

1. Ingreso

Ingrese con las credenciales autorizadas y acceda al dashboard ejecutivo.

2. Registro maestro

Use los filtros para buscar procesos por código, proveedor, institución o nivel de adquisición.

3. Expediente

Seleccione un expediente para revisar datos contractuales, checklist documental, IDD y garantías.

4. Calendario

Revise vencimientos de contratos, garantías, IDD y entregables.

5. Drive y archivos

Seleccione una carpeta local sincronizada con Google Drive para guardar respaldos o copiar archivos.

6. Reportería y Excel

Descargue PDF, Excel actualizado completo, matriz de cumplimiento y avisos por vencimiento.

7. Avisos por correo

El sistema identifica procesos por vencer o vencidos sin archivos y prepara un correo de aviso.
